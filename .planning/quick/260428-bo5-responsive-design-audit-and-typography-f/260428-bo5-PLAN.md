Apply a consistent responsive typography and padding system across all pages. The pattern is:

**Page h1 size scale (mobile-first):**
- Mobile default (≤639px): `text-3xl` (1.875rem / 30px) — safe for long Danish words at px-6
- sm (640px+): `text-4xl` (2.25rem / 36px)
- lg (1024px+): `text-5xl` (3rem / 48px)
- Class string: `text-3xl sm:text-4xl lg:text-5xl`

**Section h2 size scale (homepage sections — currently text-5xl with no override):**
- The "Til salg" h2 at `text-5xl` with no override is oversized versus "Om Laura" at `text-[28px]`. Align both section h2s to the same responsive scale as the sub-headings:
- `text-2xl sm:text-3xl lg:text-[28px]` — this ensures visual parity with "Om Laura" on all breakpoints. Wait — "Om Laura" is already `text-[28px]` fixed at all sizes. Simpler fix: make "Til salg" match it. Change `text-5xl` to `text-[28px]` (same as other h2s on this page). This achieves parity without introducing complexity.

**Horizontal padding — all page wrappers:**
The inner `div.mx-auto.max-w-screen-xl` uses `px-12 lg:px-16`. On mobile px-12 = 48px each side. Reduce to `px-6 sm:px-8 lg:px-16` across all page containers.
