**One-liner:** Server-side item preview panel on the purchase page (2-column on desktop, stacked on mobile) plus shared table-based HTML email helpers used by both inquiry actions.

## What Was Built

**Task 1 — Purchase page item preview:**

The purchase page (`/contact/purchase?piece=<slug>`) now renders a 2-column grid on desktop (`lg:grid-cols-[2fr_3fr]`): item preview on the left (4:5 aspect image + title + optional price), form on the right. On mobile it stacks — preview above, form below. When no valid slug is present the existing full-width form renders unchanged. Image path is read from Keystatic server-side; the slug is validated against the CMS reader — no client-controlled image src (T-un8-03-02 accepted).

**Task 2 — Email layout helpers:**

`email-utils.ts` now exports four helpers alongside the existing `escHtml`:

- `emailShell(content, subject)` — full `<!DOCTYPE html>` wrapper with brand header (By Blendstrup wordmark, subject as h1) and footer. Linen background (#F5F0E8), near-white card (#FDFAF5), clay border (#C4A882).
- `emailField(label, value, isLink?)` — uppercase label + value row; optional mailto link in terracotta (#A85C3A).
- `emailDivider()` — hairline clay rule.
- `emailCallout(label, value)` — left-bordered highlight box in linen with terracotta accent; used in purchase email to surface the piece title prominently.

Both `buildPurchaseEmail` and `buildCustomOrderEmail` were rewritten to use these helpers. Subject lines simplified to `Forespørgsel: <title>` and `Specialbestilling fra <name>` respectively. All user strings are still escaped — `escHtml` is called inside each helper, so call sites cannot accidentally forget to escape.
