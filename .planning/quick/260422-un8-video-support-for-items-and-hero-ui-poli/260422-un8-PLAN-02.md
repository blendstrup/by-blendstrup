**Pattern for video in cards:** When `entry.video` is set, render a `<video>` element instead of `<Image>` inside the card's `aspect-[4/5]` container. Use:
```tsx
<video
  src={entry.video}
  autoPlay
  muted
  loop
  playsInline
  className="absolute inset-0 h-full w-full object-cover transition-transform duration-500 ease-out group-hover:scale-[1.02]"
  aria-label={entry.images[0]?.alt ?? entry.title}
/>
```
`playsInline` prevents iOS from hijacking to fullscreen. `muted` is required for autoplay to work without user gesture. No `controls` attribute — these are decorative background loops.

When video is absent, fall back to the existing `<Image>` rendering (no change to existing behavior).
