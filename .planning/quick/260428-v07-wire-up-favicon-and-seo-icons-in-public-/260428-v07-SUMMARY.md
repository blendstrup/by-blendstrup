# Quick Task 260428-v07: Wire up favicon and SEO icons Summary

Wired the dropped-in favicon assets (ico, svg, 96x96 PNG, apple-touch-icon, plus 192/512 maskable PNGs) into Next.js `baseMetadata` and added a PWA `site.webmanifest` with `"any maskable"` icon purpose and brand-matching `#f5f0e8` colors.

## What was built

- Added six icon assets and `site.webmanifest` under `/public/`.
- Added `icons` + `manifest` fields to `baseMetadata` in `src/lib/metadata.ts` so Next.js renders the appropriate `<link rel="icon">`, `<link rel="apple-touch-icon">`, and `<link rel="manifest">` tags in every page's `<head>`.
- Removed the now-redundant `src/app/favicon.ico` so `/public/favicon.ico` is the single source of truth.
- Used `purpose: "any maskable"` (not `"maskable"` alone) so non-padded brand marks are not cropped on Android home screens or PWA install prompts.
- Confirmed `theme_color` + `background_color` (`#f5f0e8`) match the `--color-linen` token already declared in `src/app/globals.css`.

## Deviations from Plan

The plan assumed a working state where the icon assets, the patched `metadata.ts` (with `icons` + `manifest` fields), and `public/site.webmanifest` already existed in the worktree — only a 1-character `purpose` patch was needed. In reality the worktree was created from `master` and contained none of those: the icons sat as untracked files in the parent repo's working tree, `metadata.ts` did not yet declare `icons:` or `manifest:`, and `site.webmanifest` did not exist anywhere committed. Three deviations followed.
