<task type="auto">
  <name>Task 1: Patch site.webmanifest icon purpose to "any maskable"</name>
  <files>public/site.webmanifest</files>
  <action>
Edit `public/site.webmanifest` and change the `purpose` field on BOTH icon entries from `"maskable"` to `"any maskable"`.

Why: A manifest icon declared only as `maskable` assumes the platform will apply a safe-zone mask (typically cropping to a circle/squircle). If the icon was not designed with maskable padding, the logo gets clipped on Android home screens and some PWA install prompts. Declaring `"any maskable"` tells the browser the same asset is safe to use both as-is (any) AND as a maskable target — browsers pick the appropriate rendering path.

Leave everything else untouched:
- Keep `theme_color` = `#f5f0e8` (already matches `--color-linen` from globals.css — verified, no change needed).
- Keep `background_color` = `#f5f0e8` (same).
- Keep `name`, `short_name`, `display`, icon `src`/`sizes`/`type` fields.

Do NOT modify `src/lib/metadata.ts` — its `icons` + `manifest` block is already correctly wired.

Final file should be:
```json
{
  "name": "By Blendstrup",
  "short_name": "By Blendstrup",
  "icons": [
    {
      "src": "/web-app-manifest-192x192.png",
      "sizes": "192x192",
      "type": "image/png",
      "purpose": "any maskable"
    },
    {
      "src": "/web-app-manifest-512x512.png",
      "sizes": "512x512",
      "type": "image/png",
      "purpose": "any maskable"
    }
  ],
  "theme_color": "#f5f0e8",
  "background_color": "#f5f0e8",
  "display": "standalone"
}
```
