# Quick Task 260427-u06: Migrate Video Implementation to External Embeds

**One-liner:** Replace git-committed video file uploads with YouTube/Vimeo iframe embeds via a URL allowlist utility, and audit Keystatic GitHub storage mode readiness.

## What Was Done

### Task 1 — Schema and embed utility (commit `c0764a8`)

Created `src/lib/video-embed.ts` exporting `toEmbedUrl(raw)` which converts raw YouTube or Vimeo URLs to privacy-respecting embed URLs. Supports YouTube watch links, youtu.be short links, YouTube Shorts, and Vimeo. Returns `null` for any unrecognised URL — the iframe is never rendered for arbitrary domains (T-u06-01 security mitigation).

Replaced all four `fields.file()` video fields in `keystatic.config.ts` with `fields.url()`:
- `works.video`
- `works.mediaGallery[].video`
- `homepage.heroVideo`
- `homepage.mediaGallery[].video`

### Task 2 — Component migration (commit `52e0c65`)

Updated all video-rendering components to use `toEmbedUrl` and `<iframe>`:

- **MediaGallery**: computes `embedSrc` per item; renders iframe when valid, `bg-oat` placeholder when URL is unrecognised.
- **WorkDetail**: computes `embedSrc` at top of component body; renders iframe in primary media slot when `embedSrc` is truthy.
- **WorkCard**: removed `video` prop and branch entirely — cards always show the first image (consistent with plan decision).
- **ShopCard**: same as WorkCard — removed `video` prop and branch.
- **homepage page.tsx**: computes `heroEmbedSrc`; renders hero iframe when valid.
- **gallery/page.tsx**: removed `video` field from works mapping (WorkCard no longer accepts it).

Zero `<video>` tags remain in `src/`. TypeScript passes with zero errors.

### Task 3 — GitHub mode audit (commit `64a853c`)

Produced a structured audit report at `.planning/quick/260427-u06-migrate-video-implementation-to-external/GITHUB-MODE-AUDIT.md` covering 7 areas with PASS/WARN/FAIL verdicts.

Key findings:
- **FAIL**: Two env var name mismatches — `.env.local` uses `NEXT_PUBLIC_` prefix for `KEYSTATIC_STORAGE_KIND`, `GITHUB_REPO_OWNER`, `GITHUB_REPO_NAME`, but `keystatic.config.ts` reads the unprefixed names. The storage mode switch will never activate until these are renamed.
- **WARN**: GitHub OAuth credentials (`KEYSTATIC_GITHUB_CLIENT_ID`, `KEYSTATIC_GITHUB_CLIENT_SECRET`, `KEYSTATIC_SECRET`) are inventoried in `.env.local` but not yet provisioned.
- **PASS**: API route handler, admin page runtime, layout runtime, `generateStaticParams` compatibility — all correct.
- **PASS (with note)**: Middleware correctly gates `/keystatic` but does not block the OAuth callback path (`/api/keystatic/...`). The custom admin login must be documented for the owner.

## Decisions Made

- **Video removed from card thumbnails**: WorkCard and ShopCard now always show the first image. YouTube iframes do not support reliable autoplay in small containers; Vimeo background mode requires a paid plan for third-party domains. The video is still visible on the detail page and in media galleries.
- **YouTube-nocookie.com**: Used instead of youtube.com for embeds to avoid third-party tracking cookies (T-u06-02 mitigation).
- **MP4 files not deleted**: `public/videos/gallery/` (and potentially `public/videos/works/`, `public/videos/hero/`) still contain committed MP4 files. These should be removed only after the owner has re-hosted the videos externally and updated the CMS URL fields.

## Deviations from Plan

None — plan executed exactly as written. The `src/app/(site)/gallery/page.tsx` update was specified in the plan under Task 2 and was completed.
