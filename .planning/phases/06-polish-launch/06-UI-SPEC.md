## Copywriting Contract

Phase 6 introduces no new user-facing copy on public pages. The only new written content is the CMS training guide (`docs/cms-guide.md`), which is internal documentation — not a UI surface.

**SEO metadata copy** is new in Phase 6. These strings appear in browser tab titles, search results, and social share previews — not on-page. They must be specified here because the executor writes them into `generateMetadata`.

| Page | `title` | `description` (max 155 chars) |
|------|---------|-------------------------------|
| Root layout default | `By Blendstrup` | `Håndlavede keramikker fra By Blendstrup. Unikke stykker til salg og mulighed for at bestille specialfremstillede keramikker.` |
| Homepage (`/`) | `By Blendstrup — Håndlavet keramik` | `Oplev håndlavede keramikker skabt med omhu. Se aktuelle stykker til salg og bestil din egen specialkeramik.` |
| Gallery (`/gallery`) | `Keramik — By Blendstrup` | `Gennemse alle håndlavede keramikker fra By Blendstrup. Filtrer efter stykker til salg.` |
| Gallery detail (`/gallery/[slug]`) | `[Work title] — By Blendstrup` | `[Work description, first 120 chars, trimmed at word boundary] — By Blendstrup.` |
| Shop (`/shop`) | `Til salg — By Blendstrup` | `Keramikker til salg fra By Blendstrup. Se priser, leveringstid og kontakt for køb.` |
| Contact (`/contact`) | `Kontakt — By Blendstrup` | `Kontakt By Blendstrup for køb af keramik eller generelle spørgsmål.` |
| Purchase inquiry (`/contact/purchase`) | `Købsforespørgsel — By Blendstrup` | `Send en forespørgsel om køb af et keramikstykke fra By Blendstrup.` |

**Title format:** `[Page name] — By Blendstrup` (em dash, not hyphen). Source: Claude's Discretion (06-CONTEXT.md).

**Shared `baseMetadata` object:** Define once in `src/lib/metadata.ts`, export as `const baseMetadata`. Each page spreads it and overrides `title` and `description`. This avoids repeating `openGraph.siteName`, `robots`, and `alternates.canonical` on every page. Source: Claude's Discretion (06-CONTEXT.md).

**OG fallback image:** `/public/og-default.jpg` at 1200×630px. Referenced as `process.env.NEXT_PUBLIC_SITE_URL + '/og-default.jpg'` in `baseMetadata`. Source: D-05, D-06 (06-CONTEXT.md).

**Empty state copy:** Not applicable — Phase 6 introduces no new data-driven UI surfaces.

**Error state copy:** Not applicable — Phase 6 introduces no new forms or server actions.

**Destructive confirmation:** Not applicable — Phase 6 introduces no destructive user actions.

## Interaction Contracts

### Image Loading State (DSGN-02)

- All `next/image` instances on `WorkCard`, `ShopCard`, and `WorkDetail` must use `placeholder="blur"` with a `blurDataURL` generated by `plaiceholder`.
- The blur renders in the image's dominant colour, fading to the loaded image once the network response completes.
- No spinner, skeleton, or loading text. The blur is the only loading affordance.
- Implementation: call `getPlaiceholder(fs.readFileSync(imagePath))` in the RSC. Receive `{ base64 }`. Pass `blurDataURL={base64}` to `next/image`.
- This is a server-side build-time operation — no client JS required.

### Responsive QA Contract (DSGN-03)

Phase 6 defines the breakpoint verification matrix that the Lighthouse checkpoint plan must satisfy:

| Viewport | Width | Pages to verify |
|----------|-------|-----------------|
| Mobile | 390px (iPhone 14 simulation) | Homepage, Gallery, Work detail, Shop, Contact, Purchase form, Custom order form |
| Tablet | 768px | Homepage, Gallery, Work detail |
| Desktop | 1280px | Homepage, Gallery, Work detail, Shop |

**Pass criteria per page:**
- No horizontal scrollbar at any tested viewport width
- No text overlapping other elements
- No broken grid layouts (columns collapsing incorrectly)
- No touch targets below 44px height on mobile
- Lighthouse Performance ≥ 80 on mobile preset (Chrome DevTools)
- Lighthouse Accessibility ≥ 90 on mobile preset

**E-commerce chrome audit (DSGN-04):** During QA pass, confirm the following are absent on all pages:
- Cart icon or basket indicator
- Star rating or review count
- Stock counter or "only N left" badge
- Discount badge or percentage-off label
- "Add to cart" or "Buy now" button text

### Focus Management (unchanged from Phase 1)

- 2px solid terracotta focus ring, 2px offset, `focus-visible` only
- No changes in Phase 6

### SEO — No User-Facing Interaction

`generateMetadata`, `sitemap.ts`, and `robots.ts` produce document-level metadata only. They have no visual representation on the page. No interaction contract beyond: metadata must be present and correct in rendered HTML source for every route.
