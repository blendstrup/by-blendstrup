## Spacing Scale

Inherited in full from Phase 1. No new tokens added.

| Token | Value | Tailwind class | Phase 4 Usage |
|-------|-------|----------------|---------------|
| xs | 4px | `gap-1` / `p-1` | Icon-to-label gap in scroll indicator |
| sm | 8px | `gap-2` / `p-2` | Price/lead-time line gap inside shop card overlay |
| md | 16px | `gap-4` / `p-4` | Card caption padding, contact page item spacing |
| lg | 24px | `gap-6` / `p-6` | Card internal padding on mobile |
| xl | 32px | `gap-8` / `p-8` | Section sub-element gaps (about text block, CTA section) |
| 2xl | 48px | `gap-12` / `p-12` | Page horizontal padding (mobile), grid column gap |
| 3xl | 64px | `gap-16` / `p-16` | Page horizontal padding (desktop), about section column gap |
| 4xl | 96px | `gap-24` / `p-24` | Vertical padding top/bottom of all full-width page sections |

**Japandi rule (inherited):** Default to the larger end of the scale. Homepage sections breathe
at `py-24` (4xl). Grid gaps use `gap-8` (xl) on desktop, `gap-6` (lg) on mobile.

**Phase 4 exceptions:**
- Hero is `100vh` — no vertical padding token applies; image fills viewport edge-to-edge.
- Shop card overlay height is `30–40%` of the image area (not a spacing token — it is a proportional
  layout constraint expressed as a gradient/scrim covering approximately the bottom third).
- About section photo: `aspect-[4/5]` on desktop left column; stacks to `aspect-square` on mobile
  (avoids overly tall portrait on narrow screens).

## Typography

All roles inherited from Phase 1. The table below maps each role to Phase 4 elements.

| Role | Typeface | Size | Weight | Line Height | Tailwind class | Phase 4 Elements |
|------|----------|------|--------|-------------|----------------|-----------------|
| Display | Fraunces | 48px | 400 | 1.1 | `font-serif text-5xl font-normal tracking-tight` | Hero-adjacent section headings (Shop Preview, About, CTA) — NOT used on hero itself (no text on hero) |
| Heading | Fraunces | 28px | 400 | 1.2 | `font-serif text-[28px] font-normal tracking-tight` | Shop page title, Contact page title, About section subheadings |
| Body | DM Sans | 16px | 400 | 1.6 | `font-sans text-base font-normal` | About text, Contact page body copy, empty state copy, CTA descriptions |
| Label / UI | DM Sans | 14px | 500 | 1.4 | `font-sans text-sm font-medium` | Shop card price text, lead time text, nav labels ("Shop", "Contact"), CTA button labels, scroll indicator label |

**Price display:** Price is Label weight (500) in linen on the dark overlay. Format: numeric with
currency symbol, e.g. `1.200 kr.` (DK) or `1,200 DKK` (EN). No special display-size treatment — the
overlay is compact, Label at 14px/500 is correct.

**Lead time display:** Label weight (500) in linen at 80% opacity (`text-linen/80`) below the price.
Smaller visual weight distinguishes it from price without adding a fifth type size.

**Section headings on homepage** use Display (48px) because they are the primary typographic anchor
of each section — equivalent to page-level headings on an interior page.

**No new type sizes or weights.** The two-weight, four-size contract from Phase 1 is maintained.

## Layout Contracts

### Homepage — Section Order

Sections below the hero appear in this order (most natural for a first-time visitor — browse product,
understand context, act):

1. Hero (100vh, full-bleed image)
2. Shop Preview (up to 6 for-sale pieces, grid)
3. About (maker portrait + bilingual text)
4. Custom Order CTA (minimal — one heading, one sentence, one button)

**Rationale:** Shop Preview immediately after the hero capitalises on purchase intent at peak interest.
About follows to provide craft context and build trust. Custom Order CTA closes with the conversion
action for visitors whose needs exceed what's in stock.

### Hero Section

| Property | Value |
|----------|-------|
| Height | `100vh` — fills viewport on load |
| Width | Full bleed — no horizontal padding, no max-width constraint |
| Image treatment | `next/image` with `fill` prop, `object-cover`, `priority` (above-fold LCP), `placeholder="blur"` |
| `sizes` attribute | `"100vw"` — full viewport width at all breakpoints |
| Aspect ratio | None — image fills `100vh` container, cropping at edges |
| Text overlay | None — zero text on the hero image |
| Brand name position | Not repeated in hero; SiteHeader above carries the brand name |
| Scroll indicator | Bottom-center of hero, `absolute bottom-8 left-1/2 -translate-x-1/2` |

**Scroll indicator spec:**
- Icon: `lucide-react` `ChevronDown` at 20px, terracotta (`text-terracotta`)
- Label: "Scroll" (EN) / "Rul ned" (DA) — DM Sans 14px/500, stone color (`text-stone`)
- Layout: icon above label, `flex flex-col items-center gap-1`
- Animation: CSS-only `@keyframes bounce` at 1.5s infinite — vertical translate of 4px up/down.
  No JS animation library. Use Tailwind `animate-bounce` class (built-in).
- No click/tap handler needed — decorative. `aria-hidden="true"` on the entire indicator.

### ShopCard Component

ShopCard is a new component (not a variant of WorkCard). WorkCard links to `/gallery/[slug]`;
ShopCard links to `/gallery/[slug]` for the image/title, but adds an overlay and a separate CTA.
Sharing the overlay logic via a prop on WorkCard would couple concerns — a dedicated ShopCard is cleaner.

| Property | Value |
|----------|-------|
| Base card | Identical to WorkCard outer shell: `border border-clay bg-oat overflow-hidden` |
| Image | `next/image`, `aspect-[4/5]`, `object-cover`, `fill`. Same `sizes` as WorkCard. |
| Image hover scale | `group-hover:scale-[1.015] transition-transform duration-300 ease-out` (inherited from WorkCard) |
| Overlay scrim | `absolute bottom-0 left-0 right-0` covering bottom 35% of image. Background: linear gradient from transparent at top to `bg-ink/70` at bottom. Implementation: `bg-gradient-to-t from-ink/70 to-transparent` with `h-[40%]`. |
| Price text | Inside overlay, bottom-aligned. Label (DM Sans 14px/500) in linen (`text-linen`). Format: `1.200 kr.` (DA) / `1,200 DKK` (EN). |
| Lead time text | One line below price. Label (DM Sans 14px/500) in `text-linen/80`. Example: "Klar til levering om 2 uger" (DA) / "Ready in 2 weeks" (EN). |
| Overlay padding | `p-4` (md) inside the scrim |
| StatusBadge | Retained — top-right corner of image (terracotta pill for available, stone for sold). Since ShopPage shows only available pieces, badge will always show "available" — include it for consistency with WorkCard; remove if deemed redundant. |
| CTA — desktop | `"Kontakt for køb"` (DA) / `"Contact to buy"` (EN). Absolutely positioned inside the card, visible on hover only. Fades in: `opacity-0 group-hover:opacity-100 transition-opacity duration-200`. Position: centered over the overlay area. Button spec: see Button Spec below. |
| CTA — mobile | Always visible below the image (not inside the overlay). Rendered as a block-level element below the card, full width: `w-full`. Same button style. |
| CTA link target | `/[locale]/contact` (Phase 5 stub — the full form is Phase 5; this CTA links to contact page which has a form stub link) |
| Card title | Below card (below mobile CTA on mobile). DM Sans 14px/500 (`text-sm font-medium text-ink`). `px-3 py-2`. |

## Interaction Contracts

### Focus Management

Inherited from Phase 1: all interactive elements use `*:focus-visible` global rule — 2px solid
terracotta, 2px offset. No overrides in Phase 4.

### Link Hover

Nav links: `hover:text-ink hover:underline hover:decoration-terracotta` (inherited from "Works" link).
Body links (contact page email/Instagram): `hover:underline hover:decoration-terracotta`. Text color
stays ink — no terracotta on the text itself.

### ShopCard Hover (Desktop)

| State | Behaviour |
|-------|-----------|
| At rest | Image at natural scale. Overlay scrim always visible (contains price + lead time). CTA button `opacity-0`. |
| Hover | Image scales `scale-[1.015]`, CTA button transitions to `opacity-100`. Duration: 200ms ease-out for opacity, 300ms ease-out for scale. |
| Focus (keyboard) | CTA button becomes visible at `opacity-100` (`:focus-within` on the card wrapper or tab directly to the CTA anchor). |

### ShopCard — Mobile Always-Visible CTA

On mobile (below `sm` breakpoint), the CTA button is rendered below the image as a sibling block
element — NOT inside the overlay. The overlay still shows price + lead time. This satisfies WCAG
2.5.5 (Target Size) without relying on hover.

Implementation pattern:
```
<div class="sm:hidden w-full mt-0"> {/* always visible on mobile */}
  <a href="...">Contact to buy</a>
</div>
<div class="hidden sm:block absolute ... opacity-0 group-hover:opacity-100"> {/* hover on desktop */}
  <a href="...">Contact to buy</a>
</div>
```

### Scroll Indicator Animation

CSS-only `animate-bounce` (Tailwind built-in). No JavaScript. Progressive enhancement — if CSS
animations are disabled (prefers-reduced-motion), the indicator is still visible but static.

```css
@media (prefers-reduced-motion: reduce) {
  .animate-bounce { animation: none; }
}
```

Tailwind handles `prefers-reduced-motion` automatically via the `motion-safe:` / `motion-reduce:`
variant — use `motion-safe:animate-bounce` on the scroll indicator element.

### Hero Image Loading

- `priority` prop on the `next/image` — no lazy loading for the LCP element.
- `placeholder="blur"` with a generated `blurDataURL` from `plaiceholder` at build time.
- While loading: linen background (`bg-linen`) fills the `100vh` container.

### Active Nav Link

Use Next.js `usePathname()` (client component wrapper) to detect the current route.
Apply `text-ink underline decoration-terracotta` to the active link.
The "Works" link becomes active on `/[locale]/gallery/*` paths.
"Shop" is active on `/[locale]/shop`.
"Contact" is active on `/[locale]/contact`.
