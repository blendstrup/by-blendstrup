Update `src/app/layout.tsx` to self-host both fonts and apply CSS variable references:
```tsx
import type { Metadata } from "next";
import { Fraunces, DM_Sans } from "next/font/google";
import "./globals.css";

const fraunces = Fraunces({
  subsets: ["latin"],
  variable: "--font-serif",
  display: "swap",
  // Optical size: true enables the optical sizing axis for display use
  axes: ["opsz"],
});

const dmSans = DM_Sans({
  subsets: ["latin"],
  variable: "--font-sans",
  display: "swap",
  weight: ["400", "500"],
});

export const metadata: Metadata = {
  title: "By Blendstrup",
  description: "Handmade ceramics",
};

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html className={`${fraunces.variable} ${dmSans.variable}`}>
      <body className="bg-linen text-ink font-sans antialiased">{children}</body>
    </html>
  );
}
```

Note: `--font-serif` and `--font-sans` variable names in next/font declarations must match the @theme token names exactly so Tailwind's `font-serif` and `font-sans` utilities resolve to the Google Fonts. The `variable` option on each font declaration creates a CSS custom property that Tailwind v4 reads from the @theme block.

Do NOT create a `tailwind.config.ts` or `tailwind.config.js` — Tailwind v4 uses the CSS-native `@theme` block exclusively. No JS config file.
