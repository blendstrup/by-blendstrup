Create `keystatic.config.ts` at project root — environment-based storage switching:
```typescript
import { config, singleton, fields } from "@keystatic/core";

// Storage mode: 'local' for dev, 'github' for production
// Driven by KEYSTATIC_STORAGE_KIND env var (defaults to 'local')
const storage =
  process.env.KEYSTATIC_STORAGE_KIND === "github"
    ? ({
        kind: "github",
        repo: {
          owner: process.env.GITHUB_REPO_OWNER!,
          name: process.env.GITHUB_REPO_NAME!,
        },
      } as const)
    : ({ kind: "local" } as const);

export default config({
  storage,
  ui: {
    brand: { name: "By Blendstrup" },
  },
  singletons: {
    settings: singleton({
      label: "Site Settings",
      path: "content/settings",
      format: { data: "yaml" },
      schema: {
        siteTitle: fields.text({
          label: "Site Title",
          description:
            "The name that appears in browser tabs and search results.",
          defaultValue: "By Blendstrup",
          validation: { length: { min: 1, max: 60 } },
        }),
      },
    }),
  },
});
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser→/keystatic | Owner browser accesses Keystatic admin UI |
| keystatic→GitHub | Keystatic Cloud writes commits to the repo via GitHub OAuth |
| GitHub→Vercel | Vercel webhook triggers deploy from GitHub push |
| env→keystatic.config | KEYSTATIC_STORAGE_KIND and repo credentials read from environment |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Elevation of Privilege | /keystatic admin UI | mitigate | Keystatic Cloud provides its own OAuth-based authentication for the GitHub storage path. In local dev, /keystatic is protected by being localhost-only. In production, Keystatic's auth flow prevents unauthorized access. Do NOT add application-level auth middleware that would conflict with Keystatic's own auth. |
| T-03-02 | Information Disclosure | .env.local / Vercel env vars | mitigate | KEYSTATIC_GITHUB_CLIENT_SECRET and KEYSTATIC_SECRET must never be committed. .env.local is gitignored. .env.local.example contains no values, only key names. |
| T-03-03 | Tampering | GitHub commits via Keystatic | accept | Only authenticated owner can trigger Keystatic saves. GitHub commit history provides full audit trail. Vercel preview builds show changes before production merge if desired. |
| T-03-04 | Denial of Service | content/settings.yaml | accept | YAML file is minimal (one field). No user-supplied content in Phase 1. Low risk. |
</threat_model>
