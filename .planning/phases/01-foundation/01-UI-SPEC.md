## Design System

| Property | Value |
|----------|-------|
| Tool | none (shadcn/ui explicitly rejected â€” see CLAUDE.md) |
| Preset | not applicable |
| Component library | Radix UI primitives only, pulled individually as needed |
| Icon library | lucide-react |
| Font | Fraunces (serif display) + DM Sans (humanist sans body) via next/font |

**Font rationale:** Fraunces is a variable optical-size serif with warmth that suits handmade craft; DM Sans is clean and neutral at body sizes. Both are Google Fonts, self-hosted via `next/font` for zero CLS. Alternative: Cormorant Garamond (serif) + Inter (sans) â€” acceptable substitution with identical implementation pattern.

## Spacing Scale

Declared values (multiples of 4 only). Tailwind v4 `@theme` maps these as CSS custom properties:

| Token | Value | Tailwind class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `gap-1` / `p-1` | Icon gaps, inline label padding |
| sm | 8px | `gap-2` / `p-2` | Tight element separation |
| md | 16px | `gap-4` / `p-4` | Default component spacing |
| lg | 24px | `gap-6` / `p-6` | Card internal padding |
| xl | 32px | `gap-8` / `p-8` | Section sub-element gaps |
| 2xl | 48px | `gap-12` / `p-12` | Section padding (horizontal page margins, mobile) |
| 3xl | 64px | `gap-16` / `p-16` | Section padding (horizontal page margins, desktop) |
| 4xl | 96px | `gap-24` / `p-24` | Major vertical section breaks |

**Japandi rule:** Default to the larger end of the scale. Prefer `p-12`/`p-16` over `p-6` for section padding. Prefer `gap-8`/`gap-12` over `gap-4` for grid gaps. Whitespace is the primary design element.

Exceptions:
- Language toggle touch target: minimum 44px height (accessibility â€” meets WCAG 2.5.5 AA)
- Header height: 64px (aligns to 3xl token)
- **`--spacing-4xl: 96px` â€” out-of-set exception:** 96px is used exclusively for major vertical section breaks (top/bottom padding of full-width sections, and vertical rhythm between distinct page regions). Japandi requires generous vertical breathing room; 64px (3xl) is insufficient at section boundaries where the eye must perceive a clear pause. 96px is 3xl Ă— 1.5, maintains the 4px grid, and is not used for intra-component spacing. This is the only value that exceeds the standard seven-step set.

## Typography

Two typefaces, four sizes, two weights. No exceptions in Phase 1.

| Role | Typeface | Size | Weight | Line Height | Tailwind class |
|------|----------|------|--------|-------------|----------------|
| Body | DM Sans | 16px | 400 (regular) | 1.6 | `text-base font-normal` |
| Label / UI | DM Sans | 14px | 500 (medium) | 1.4 | `text-sm font-medium` |
| Heading | Fraunces | 28px | 400 (regular) | 1.2 | `text-[28px] font-normal` |
| Display | Fraunces | 48px | 400 (regular) | 1.1 | `text-5xl font-normal` |

**Weights in use:** 400 (regular) and 500 (medium) only. No bold (700) in Phase 1 â€” Japandi restraint avoids heavy weight contrast. Fraunces optical-size variation creates hierarchy through size alone, not weight.

**Letter-spacing:** Display and Heading use `tracking-tight` (-0.01em). Body and Label use default tracking.

**Serif display usage:** Fraunces is reserved for headings and display text only. All UI chrome (nav labels, buttons, form labels, captions) uses DM Sans.

## Color

Tailwind v4 `@theme` block declares these as CSS custom properties. The default Tailwind palette is entirely overridden â€” no blue-500, red-400, etc. accessible to developers.

| Role | Token name | Hex | Usage |
|------|-----------|-----|-------|
| Dominant (60%) | `--color-linen` | `#F5F0E8` | Page background, primary surface |
| Secondary (30%) | `--color-oat` | `#EDE7D9` | Header/nav background, card surfaces, sidebar |
| Neutral mid | `--color-clay` | `#C4A882` | Borders, dividers, subtle decoration |
| Neutral dark | `--color-stone` | `#7A6A58` | Secondary text, captions, placeholder text |
| Foreground | `--color-ink` | `#2C2418` | Primary body text, headings |
| Accent (10%) | `--color-terracotta` | `#A85C3A` | Reserved elements only (see below) |
| Destructive | `--color-fault` | `#8B3A2A` | Destructive actions only |
| Surface inverse | `--color-ink-surface` | `#1E1A12` | Footer background (dark anchor) |

**60/30/10 split in practice:**
- 60% linen: `<body>` background, all page-level backgrounds
- 30% oat: `<header>`, card backgrounds, any panel or sidebar
- 10% terracotta: accent-reserved elements only

**Accent (terracotta) reserved for:**
1. Language toggle active indicator (the currently selected locale)
2. Focus ring outline on interactive elements (keyboard navigation)
3. No other uses in Phase 1 â€” future phases may extend this list

**Destructive (fault):** Not used in Phase 1 â€” no destructive actions exist in the foundation scaffold.

## Interaction Contracts

### Language Toggle (I18N-01)

- Rendered in the site header as a text control, not a dropdown.
- Pattern: two locale codes side by side, separated by a hairline divider: `DA | EN`
- Active locale: terracotta (`--color-terracotta`) text at 500 weight.
- Inactive locale: stone (`--color-stone`) text at 400 weight, with underline on hover.
- Touch target: minimum 44px height.
- On activation: `router.push()` to the matching locale path â€” full page navigation, no client-side partial update.
- No animation in Phase 1 â€” that is addressed in Phase 6.

### Focus Management

- All interactive elements show a 2px solid terracotta focus ring with 2px offset (`outline: 2px solid var(--color-terracotta); outline-offset: 2px`).
- No `:focus` styles suppressed â€” only `:focus-visible` used so mouse users do not see rings.

### Skip Link (a11y)

- First focusable element in the DOM.
- Visually hidden until focused, then appears at top-left of viewport.
- Links to `#main-content`.
