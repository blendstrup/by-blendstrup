<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: DNS setup — Resend domain verification + SPF/DKIM/DMARC records</name>
  <what-built>
Claude has:
- Installed the Resend SDK and wired it to both server actions
- Created .env.local.example documenting all required env vars
- Written the server actions to use RESEND_API_KEY, RECIPIENT_EMAIL, and RESEND_FROM_ADDRESS env vars

What requires human action: Adding DNS records to the owner's domain and verifying the
domain in the Resend dashboard. This cannot be automated — DNS provider access is required.
  </what-built>
  <how-to-verify>
Complete these steps in order:

**Step 1: Create a Resend account (if you don't have one)**
- Go to resend.com and sign up for a free account

**Step 2: Create an API key**
- Resend Dashboard → API Keys → Create API Key
- Copy the key — you will only see it once
- Add it to .env.local as RESEND_API_KEY=re_...
- Also add it to Vercel: Dashboard → [project] → Settings → Environment Variables

**Step 3: Add your domain in Resend**
- Resend Dashboard → Domains → Add Domain
- Enter the owner's domain (e.g. byblendstrup.dk)

**Step 4: Add DNS records at your DNS provider**
Resend will show you three records to add. They look like this:

| Type | Name | Value |
|------|------|-------|
| TXT | send.[your-domain] | v=spf1 include:amazonses.com ~all |
| CNAME or TXT | resend._domainkey.[your-domain] | [value from Resend dashboard] |
| MX | send.[your-domain] | feedback-smtp.us-east-1.amazonses.com (priority 10) |

Add all three records at your DNS provider (Cloudflare, Namecheap, etc.)

**Step 5: Add DMARC record (separate from Resend — owner-added)**
At your DNS provider, add:

| Type | Name | Value |
|------|------|-------|
| TXT | _dmarc | v=DMARC1; p=none; rua=mailto:[owner-email] |

Note: Start with p=none to monitor. Upgrade to p=quarantine after confirming SPF/DKIM pass.

**Step 6: Wait for DNS propagation (up to 48 hours, usually 15-30 minutes)**
Then verify in Resend Dashboard → Domains → [your domain] → click Verify.
Status should change to "Verified".

**Step 7: Set remaining env vars**
- .env.local: set RECIPIENT_EMAIL=[owner email], RESEND_FROM_ADDRESS=noreply@[your verified domain]
- Vercel project settings: add RECIPIENT_EMAIL and RESEND_FROM_ADDRESS

**Step 8: Verify DNS propagation with dig commands**
```bash
dig TXT send.[your-domain]          # should show v=spf1 include:amazonses.com
dig TXT resend._domainkey.[your-domain]   # should show DKIM key
dig TXT _dmarc.[your-domain]        # should show v=DMARC1
```
  </how-to-verify>
